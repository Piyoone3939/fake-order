# ゲーム仕様書：Fake Order

## 概要

| 項目 | 内容 |
|------|------|
| ジャンル | 非対称対戦 / 情報戦略アクション |
| プレイ人数 | 1vs1（オンライン対戦） |
| プラットフォーム | PC（Unity製） |
| 1試合の目安時間 | 10〜15分 |

---

## コンセプト

「誰が誰を騙しているか分からない」状態を作り出す非対称対戦ゲーム。

攻め側（スパイ）は組織に潜入して情報を収集し脱出する。
守り側（オーガナイザー）は偽情報を使ってスパイを罠に誘導するか、正体を見破って排除する。

両者が互いに騙し合うことで、既存の「隠れる vs 見つける」を超えた
**信頼と疑念のジレンマ**が生まれる。

---

## プレイヤーロール

### スパイ（攻め側）
- 視点：一人称（FPS視点）
- 目標：施設内の3箇所の情報端末からデータを収集し、脱出ポイントへ到達
- 初期情報：施設の大まかな間取りのみ（警備配置・カメラ位置は不明）
- 特殊能力：命令書偽造、端末ハッキング、変装

### オーガナイザー（守り側）
- 視点：管理ルームから認証後、選択式の監視カメラ映像へ接続（俯瞰による全階透視はしない）
- 目標：スパイを罠にはめるか、正しく識別して排除
- 初期情報：NPCの配置と行動ルーティン（ただし全カメラはリアルタイムでない）
- 特殊能力：命令発令、偽情報リーク、警備強化

## コアシステム

---

### システム1：命令書偽造（Forge Order）

#### 概要
スパイは施設内の端末から偽の命令書を発行し、NPCの行動を操作できる。
守り側は命令ログを監視して異常を検知する。

#### スパイ側の操作
- 端末にアクセス（5秒のハッキング時間）
- 命令の種類を選択して発行
  - 情報漏洩：特定エリアの情報を取得する 
  - 警備移動命令：指定エリアの警備NPCを別エリアへ移動
  - 点検命令：NPCを特定の場所に集合させる
  - 緊急命令：一時的にカメラ監視を中断させる

#### 守り側の対抗手段
- 命令ログ画面で直近の命令一覧を確認できる
- 不審な命令を「無効化」できる（5秒の操作時間が必要）
- 命令ログを確認する操作は**スパイ側に「ログ確認中」として通知される**

#### バランス制約
- スパイが同一エリアに偽造命令を3回以上出すと守り側に**警告アラート**が出る
- 守り側が偽造を無効化すると、そのエリアは5分間命令受け付け不可になる

---

### システム2：遅延監視（Delayed Surveillance）

#### 概要
監視カメラはエリアによって映像の遅延時間が異なる。
守り側は「今」ではなく「過去」を見ながら判断する必要がある。

#### エリアごとの遅延設定

| エリア | 遅延時間 | 備考 |
|--------|---------|------|
| 一般通路 | 20秒 | 最もスパイが通る場所 |
| 一般執務室 | 10秒 | 情報端末が置かれる |
| 重要区画（金庫室周辺） | リアルタイム | ただしカメラが1台のみ |
| 外周・搬入口 | 45秒 | 脱出ポイント付近 |
| 盲点エリア | カメラなし | 施設に2〜3箇所存在 |

#### 守り側のUI
- 各カメラ映像には「現在時刻 - 遅延時間」がタイムスタンプで表示される
- 守り側はカメラ映像と俯瞰マップの**NPCリアルタイム位置**を照合して推理する
  - 俯瞰マップ上のNPCはリアルタイムで動いている（スパイかどうかは分からない）
  - カメラ映像は遅延あり

#### 容疑者照合・排除

- 選択中の遅延映像に対して3秒間の身元照合を実行できる。
- 判定には現在のスパイ位置ではなく、その遅延フレームが撮影された時点の所在記録を使用する。
- 映像内にスパイが存在した場合は照合成功となり、スパイ排除でゲームを終了する。
- 誤認すると認証ポイントを1消費し、10秒間は再照合できない。認証ポイントは1試合3点で、0になると照合権限を失う。
- 長押し中にキーを離した場合は照合を中止し、ポイントを消費しない。

#### スパイ側の知識
- スパイはゲーム開始時に各エリアの遅延時間を知らない
- 端末ハッキングで遅延情報を取得できる（情報収集の一種として機能）

---

### システム3：情報鮮度（Information Freshness）

#### 概要
スパイが収集する情報には「鮮度」があり、時間経過で劣化する。
守り側は意図的に「古い情報」を新鮮に見せかけてスパイを罠に誘導できる。

#### 情報の状態

| 状態 | 経過時間 | スパイ画面の表示色 | 内容の正確性 |
|------|---------|-----------------|------------|
| 新鮮 | 0〜60秒 | 青 | 正確 |
| 劣化 | 60〜180秒 | 黄 | 警備位置などに誤差あり |
| 腐敗 | 180秒以上 | 赤 | 大部分が古く使えない |
| **罠** | 任意 | **青（偽）** | 守り側が仕掛けた偽情報 |

#### 罠情報の仕掛け方（守り側）
1. 守り側は「情報端末」を選択して偽データを流し込む（10秒の操作）
2. 偽データは「新鮮」に見せかけた青色でスパイ画面に表示される
3. 内容例：
   - 警備が撤退したと見せかけて実は配置済み
   - 脱出ルートが開放されていると見せかけて封鎖済み
   - 目標端末の場所を1つズラして表示

#### 識別ヒント（スパイ側）
- 同じ情報を2つの端末から取得すると「照合」ができる
- 照合で矛盾があれば「要注意」マーカーが付く
- ただし照合には追加の時間とリスクがかかる

## 施設マップ設計

### 全体構成
- 3階建て構造
- 各フロアに情報端末を複数配置
- スパイは3箇所から情報収集 → 脱出ポイントへ

```
[ 3F ] 経営・指令・監視中枢・最高機密区画
[ 2F ] 重要データ・監視分析・記録保管区画
[ 1F ] 一般執務・来客・警備・搬入区画（脱出ポイント）
```

### 3階建てプロトタイプ詳細

| 階 | 部署・区画 | 認証区画 |
|---|---|---|
| 1F | エントランス、ロビー、オープンオフィス、休憩室、倉庫、会議室、警備室、東側オフィス | 2Fパス保管室 |
| 2F | ロビー、オープンオフィス、警備分析室、データルーム、監視室、記録保管室、会議室、個室オフィス | 3Fパス保管室 |
| 3F | ロビー、役員室、管理室、司令室、監視中枢、資料保管室 | メインサーバー室、マスターキー保管室 |

- 各階は中央エレベーターとロビーを動線の核にし、外周の部屋、分岐通路、備品や間仕切りで直線の視界を切る。
- 平面寸法は初期複雑フロア版の約1.4倍とし、3階を通した移動・探索に十分な距離を持たせる。
- 執務区画にはデスク、椅子、植栽と操作不能なブラフPCを多数配置し、3台の情報端末だけが目立たない構成にする。
- 認証区画はドアの解析を完了するまで入室できない。
- 階移動はエレベーターパネルを使い、移動先の階は同時に見通せない。
- 監視カメラは天井付近の部屋角または通路端からの透視映像とし、真上からの俳瞰映像は使わない。
- オーガナイザーの俳瞰マップは1Fのみ表示し、2F・3Fは限定された監視映像から状況を推測する。

### 動線設計の原則
- 最短ルートは必ずカメラに映る
- 遠回りルートはカメラの死角を通るが時間がかかる
- 守り側が命令でNPCを動かすことで動線を変化させられる

## 勝利条件

### スパイの勝利
- 3箇所の情報端末から**新鮮な情報**を収集し、脱出ポイントへ到達

### オーガナイザーの勝利
- 遅延監視映像からスパイを正しく識別して排除（誤認は認証ポイント-1）
- 制限時間（15分）以内にスパイの侵入を阻止（脱出を防ぐ）

### 引き分け
- スパイが情報を1〜2箇所しか収集できずに脱出した場合→部分成功（引き分け）

## 疑惑システム（サポートシステム）

スパイには常に「疑惑ゲージ」が蓄積される。
現在のローカル縦切りでは、ゲージが満タンになると警備に身元を特定され、スパイ排除でゲーム終了となる。

### ゲージが上昇する行動

| 行動 | 上昇量 |
|------|-------|
| 走る | 大 |
| 制限エリアへの侵入 | 大 |
| 同じ場所に長時間滞在 | 中 |
| NPCとのルーティンの不一致 | 中 |
| カメラを見上げる | 小 |

### 現在の全階NPCシステム

- 1Fは一般社員6人・警備員2人、2Fは一般社員6人・警備員2人、3Fは職員4人・警備員3人が、それぞれ独立したNavMesh上を巡回する。
- 社員は距離9m・視野角70度、警備員は距離13m・視野角90度を基準にスパイを検知する。
- 壁や家具で視線が遮られている場合は検知しない。
- 発見中の走行は1.8倍、端末ハッキングは2.2倍の疑惑補正を受ける。
- 警備移動命令は同じ階の警備員、点検命令は同じ階の一般NPC、緊急命令は同じ階の全NPCを発行端末周辺へ誘導する。
- 命令行動は最大35秒間続き、オーガナイザーが命令を無効化すると即座に通常巡回へ戻る。情報漏洩命令ではNPCは移動しない。
- 1FのNPCは俯瞰マップと監視カメラ、2F・3FのNPCは各階の監視カメラから確認できる。

### 現在の社員擬態システム

- スパイにも一般社員と同じカプセル体型、制服、肌色の3D外見を持たせ、監視映像だけでは外見から即断できないようにする。
- 社員とスパイは4種類の共通外見バリエーションを使用し、スパイの制服と偽装社員IDは試合ごとに変更する。
- 左`Ctrl`による社員歩行は一般社員のNavMesh移動速度2.1m/sと一致する。
- 社員の近くで社員歩行を続けると疑惑の自然減衰が加速する。通常速度でNPCの視界へ入ると疑惑補正が1.25倍、走行時は1.8倍となる。
- 88台のブラフPCは4秒間の通常業務地点として利用でき、1F休憩室には5秒間の休憩地点を置く。完了時に疑惑を減少させる。

### オーガナイザー検証用スパイAI

- ローカル検証でオーガナイザーを選択した場合のみ、社員と同じ外見のスパイAIをメインフェーズ中に有効化する。
- NavMesh上を社員速度2.1m/sで移動し、1F→2F→3Fの情報端末を5秒ずつハッキングする。
- 施錠区画ではアクセス解除状態を経て独立NavMeshへ移り、収集後はエレベーター経由で次階へ進む。
- 3件収集後は1Fの脱出地点へ向かう。身元照合による排除、ロール選択への戻り、または決着時に停止する。
- このAIは監視ゲームループの単体テスト用であり、オンライン対戦実装後は実プレイヤーに置き換える。
- スパイ自身の外見は一人称カメラからだけ除外し、室内監視カメラには描画する。

### ゲージが下がる行動
- NPCに混じって歩く
- 業務端末を操作する（偽造でなくても）
- 休憩室など自然な場所に滞在する

## UI設計

### スパイ（攻め側）
- 画面上部：収集済み情報のステータス（鮮度付き）
- 画面右上：疑惑ゲージ
- ミニマップ：収集した情報だけ反映（不完全な地図）
- インタラクションプロンプト：端末・NPCへのアクション

### オーガナイザー（守り側）
- 開始画面：自由移動できる管理ルーム内の一人称視点
- 監視端末：机のモニターへ接近して操作した時だけ認証・監視UIを表示し、離れると閉じる
- メイン画面：選択中の監視カメラを表示する16:9の大画面
- サブ画面：選択可能なカメラ映像グリッド（最大6画面、遅延タイムスタンプ付き）
- 命令ログ：直近20件の命令一覧
- アラートパネル：疑惑通知・ログ異常通知
- 管理ルーム設備：作戦卓、通信コンソール、証拠保管庫、緊急制御盤（後続システムを追加するための配置土台）

## ゲームフロー

```
1. ロール選択（スパイ or オーガナイザー）
      ↓
2. 準備フェーズ（60秒）
   - スパイ：潜入口の選択、初期変装の選択
   - オーガナイザー：NPCルーティン設定、罠情報の初期配置
      ↓
3. メインフェーズ（最大15分）
   - 情報収集・偽造命令・遅延監視・鮮度管理が同時進行
      ↓
4. 決着
   - 脱出成功 / 排除 / 時間切れ
      ↓
5. リザルト画面
   - 両者の行動ログが公開される（何が罠で何が本物だったか）
```

## 開発フェーズ案

### Phase 1：プロトタイプ（〜2ヶ月）
- ローカル1vs1でコアシステム3つを実装
- マップ：1フロアのみ、NPCなし
- 目標：B/C/Eの駆け引きが成立するか検証

### Phase 2：α版（〜4ヶ月）
- NPCと疑惑ゲージの実装
- 3フロアマップ完成
- ローカルで完全なゲームループが動く状態

### Phase 3：β版（〜6ヶ月）
- オンライン対戦実装
- UI/UXの整備
- バランス調整
