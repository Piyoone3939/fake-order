# Fake Order 操作方法

この文書は、現在の Unity ローカルプロトタイプの実装に対応した操作ガイドです。

## 起動方法

1. Unity Hub から `FakeOrder` フォルダーを Unity 6000.4.1f1 で開きます。
2. Project ウィンドウから `Assets/Scenes/GamePrototype.unity` を開きます。
3. 初回のスクリプト読み込みが終わるまで待ち、Hierarchyに `RoleSelectionUI`、`Spy`、`Organizer`、`SpyUI`、`OrganizerUI` などが表示されたことを確認します。
4. `Ctrl + S` で、配置されたオブジェクトをシーンへ保存します。
5. Unity Editor 上部の Play ボタンを押します。
6. Play モードになったら `Game` タブを選択します。

旧状態のシーンでは最初のHierarchyに `Main Camera`、`Directional Light`、`GameObject` だけが表示されます。Playモードを停止した状態でスクリプトの再読み込みが完了すると、`GameObject` の `GamePrototypeSetup` が編集用オブジェクトを一度だけ配置します。それ以降はPlay開始時に作り直されないため、通常のシーンオブジェクトとして編集できます。

## プロトタイプの概要

- ローカル検証に加え、同じLAN内の2台をHost／Clientとして接続する基盤を利用できます。
- 開始後は60秒間の準備フェーズがあり、その後メインフェーズへ移行します。
- スパイは異なる3台の端末から新鮮な本物の情報を収集し、脱出ポイントから脱出すると勝利です。
- 情報が1～2件の状態で脱出すると部分成功、情報なしで脱出すると失敗になります。
- メインフェーズの制限時間は15分です。

## ロール選択

Playすると最初に透過PNGのタイトルロゴ画面が表示されます。`Enter`または画面クリックでロール選択へ進みます。ロゴは `RoleSelectionUI/TitlePanel/LogoRoot/LogoImage` に生成され、フェード、スキャンライン、発光、控えめなグリッチが重なります。

| ロール | 操作 |
|---|---|
| タイトルから進む | `Enter`、または画面をクリック |
| スパイを選択 | 画面の「スパイ」をクリック、または `1` |
| オーガナイザーを選択 | 画面の「オーガナイザー」をクリック、または `2` |
| ロール選択へ戻る | ゲーム中に `F1` |

選択後は60秒の準備フェーズが始まり、画面上部に残り時間が表示されます。メインフェーズが始まると、選択したロールの入力だけが有効になります。
準備を省略してすぐ確認したい場合は、準備フェーズ中に `Enter` を押します。

## LAN接続（Phase 3.1）

2台のPCを同じLANへ接続し、両方で同じバージョンのゲームを起動します。画面上部の`ONLINE (LAN)`パネルを使います。

1. 監視側のPCで`HOST`を押します。HostはUDPポート`7777`で待機します。
2. 潜入側のPCで`HOST IP`欄へHostのLAN内IPv4アドレスを入力し、`JOIN`を押します。
3. 2人の接続準備が完了すると、Hostへオーガナイザー、Clientへスパイが自動割り当てされます。
4. 準備フェーズの開始操作はHostが行い、ClientはHostの進行を受信します。

Windows Defenderファイアウォールの確認が出た場合は、プライベートネットワークの通信を許可してください。接続を切るときは`DISCONNECT`、従来の1台検証は`OFFLINE`を押します。

Phase 3.1で同期されるのは、接続状態、ロール、準備／メイン／終了フェーズ、残り時間、勝敗結果です。プレイヤー移動や端末・NPC状態の完全同期は次フェーズで実装します。

## スパイ側の操作

| 操作 | キー／入力 |
|---|---|
| 移動 | `W` `A` `S` `D` または矢印キー |
| 視点移動 | マウス移動 |
| 走る | 移動中に左 `Shift` |
| 社員と同じ速度で歩く | 移動中に左 `Ctrl` |
| 端末・脱出ポイントを操作 | 対象を見ながら `E` |
| 認証ドアをハッキング | ドアを見ながら `E`、完了まで近くで待機 |
| 階を移動 | エレベーターパネルを見ながら `E` |
| マウスカーソルのロック／解除 | `Esc` |

走り始めると疑惑ゲージが上昇します。操作可能な対象を見ると、画面下部に `[E]` のプロンプトが表示されます。

### 社員への擬態

- スパイは監視カメラ上で一般社員と同じ体型・制服に見えます。試合開始ごとに偽装社員IDと制服色が変わり、画面上部へ現在のIDが表示されます。
- 左`Ctrl`を押しながら移動するとNPC社員と同じ速度になります。社員の近くでこの速度を保つと疑惑が通常より速く減少します。
- 左`Ctrl`を使わない通常移動や走行は、NPCに発見された際の疑惑上昇が速くなります。
- ブラフPCを見て`E`を押すと4秒間の通常業務、1F休憩室の設備では5秒間の休憩を行います。完了すると疑惑が減少します。
- 業務・休憩中に`E`を押すと中断できます。中断時は疑惑が減少しません。

### NPCと警備員

- 1Fには一般社員6人・警備員2人、2Fには一般社員6人・警備員2人、3Fには職員4人・警備員3人が巡回しています。
- NPCの正面視界へ入ると疑惑が徐々に上昇します。壁、間仕切り、家具で視線を切ると発見状態が解除されます。
- 発見中に走る、または端末をハッキングすると疑惑の上昇が速くなります。
- NPCがオレンジ／赤へ変化すると、スパイを認識している状態です。
- 疑惑が100%になるとスパイ排除となり、ゲームが終了します。

### フロア構成と移動

施設は中央エレベーターで接続された3階構成です。各階の壁と部屋はシーン内に編集可能なオブジェクトとして配置されています。

| 階 | 主な区画 | ハッキングが必要な区画 |
|---|---|---|
| 1F | エントランス、ロビー、オープンオフィス、会議室、警備室、倉庫 | 2Fパス保管室 |
| 2F | 警備分析室、データルーム、監視室、記録保管室、複数オフィス | 3Fパス保管室 |
| 3F | 役員室、管理室、司令室、監視中枢、資料保管室 | メインサーバー室、マスターキー保管室 |

- エレベーター前のパネルへ照準を合わせて`E`を押すと、対応する階のロビーへ移動します。
- 認証扉は照準を合わせて`E`を押し、解析が完了するまで扉の近くで待ちます。
- 脱出地点は1Fエントランス側にあります。
- 俯瞰マップに表示されるのは1Fだけです。2F・3Fは室内角や通路端の監視カメラ映像から把握します。
- オフィス内の大半のモニターは、情報を持たないブラフPCです。ブラフPCでは「通常業務」、青い画面の3台では情報収集・命令偽造を行えます。
- ブラフPCのデスク、椅子、植栽にも当たり判定があるため、デスク島の間の通路を使って移動します。

### 端末操作

端末を見ながら `E` を押し、表示されたメニューから次の操作を選びます。

| 操作 | キー |
|---|---|
| 情報収集 | `1` |
| 命令書偽造：情報漏洩 | `2` |
| 命令書偽造：警備移動命令 | `3` |
| 命令書偽造：点検命令 | `4` |
| 命令書偽造：緊急命令 | `5` |
| メニューをキャンセル | `E` |

- 選択後、ハッキング完了まで約5秒かかります。
- ハッキング中またはメニュー表示中に端末から離れすぎると、自動的にキャンセルされます。
- 同じ端末を再度収集すると情報の鮮度は更新されますが、収集件数は増えません。
- 収集した情報は画面上部に表示され、時間経過に応じて鮮度が変化します。
- 罠情報は取得時点では本物と同じ表示になるため、UIの色だけでは判別できません。
- 収集済み端末はミニマップに追加されます。
- 警備移動命令（`3`）は同じ階の警備員、点検命令（`4`）は同じ階の一般NPC、緊急命令（`5`）は同じ階の全NPCを発行端末周辺へ誘導します。
- 誘導は最大35秒間です。オーガナイザーが対象命令を無効化すると、NPCは途中でも通常巡回へ戻ります。情報漏洩命令（`2`）ではNPCは移動しません。

### 脱出

施設内の `ExitPoint` を見ながら `E` を押します。脱出結果は、収集した「新鮮かつ罠ではない情報」の数によって決まります。

| 情報数 | 結果 |
|---|---|
| 3件以上 | スパイ勝利 |
| 1～2件 | 部分成功 |
| 0件 | 失敗 |

## オーガナイザー側の操作

オーガナイザーを選ぶと、管理ルーム内の一人称視点から始まります。開始直後に認証画面は表示されません。部屋を歩いて机の監視端末へ近づき、モニターへ照準を合わせて`E`を押すと認証画面が開きます。

メインフェーズでは、オーガナイザーの監視テスト用に社員へ擬態したスパイAIが侵入します。AIは1F→2F→3Fの情報端末を順番に5秒間ハッキングし、3件を収集すると1Fの脱出地点へ向かいます。監視端末を開いてカメラを切り替え、不自然な行動を見分けてください。

| 操作 | キー／入力 |
|---|---|
| 管理ルーム内を移動 | `WASD` または矢印キー |
| 視点を動かす | マウス |
| 監視端末を開く | モニターを見て `E` |
| 監視システムをロック解除 | `AUTHENTICATE` をクリック、または `Enter` |
| 右側のカメラを左の大画面に表示 | カメラ映像をクリック |
| 次のカメラを左の大画面に表示 | `C` |
| 選択映像の容疑者を照合・排除要請 | `R`を3秒間長押し |
| 監視端末を閉じる | `E` または `Esc` |
| 命令ログを開く／閉じる | `L` |
| 罠情報の配置メニューを開く／閉じる | `T` |

監視端末を開いている間もWASDで後退できます。端末から一定距離離れると監視画面は自動的に閉じ、管理ルーム視点へ戻ります。認証状態はその試合中だけ保持されるため、再び端末へ近づけば認証画面を省略して監視画面を開けます。

監視カメラにはエリア名、設定された遅延時間、映像時刻が表示されます。右側で選択した映像は青く強調され、同じ遅延映像が左側へ16:9の大画面で表示されます。

### 容疑者照合・排除

1. 右側の一覧または`C`で、調べたいカメラを左の大画面へ表示します。
2. スパイだと判断した映像で`R`を3秒間長押しします。途中で離すとキャンセルされます。
3. 映像の撮影時点でスパイがその監視エリアにいた場合、照合成功となりスパイを排除します。

- 判定対象は現在位置ではなく、画面に表示中の遅延フレームです。表示された映像時刻を確認してください。
- 誤認すると認証ポイントを1点失い、10秒間は再照合できません。
- 認証ポイントは1試合3点です。0になると、その試合では照合・排除要請を使用できません。
- 準備フェーズ中は照合できません。メインフェーズ開始後に利用できます。

### 命令ログ

`L` で命令ログを開いた後、次のキーで操作します。

| 操作 | キー |
|---|---|
| 上のログを選択 | `I` |
| 下のログを選択 | `K` |
| 選択した命令を無効化 | `Enter` またはテンキー `Enter` を5秒間長押し |

- 無効化の長押し中は進捗バーが表示されます。途中でキーを離すと進捗はリセットされます。
- 命令を無効化すると、そのエリアは5分間、偽造命令を受け付けなくなります。
- オーガナイザーが命令ログを開いている間、スパイ側には「命令ログを確認中」と通知されます。
- 同一エリアで偽造命令が3回発行されるごとに、オーガナイザー側へ警告が表示されます。

### 罠情報の配置

罠情報は準備フェーズ中またはメインフェーズ中に配置できます。

1. `T` で罠情報配置メニューを開きます。
2. `1`、`2`、`3` のいずれかで対象端末を選択します。
3. そのまま10秒待つと配置が完了します。

| 操作 | キー |
|---|---|
| 端末 #1を選択 | `1` |
| 端末 #2を選択 | `2` |
| 端末 #3を選択 | `3` |
| メニューを閉じる／配置をキャンセル | `T` または `Esc` |

- 配置中は進捗バーが表示されます。
- 配置済みの端末は選択メニューに表示されます。
- 配置を完了した端末からスパイが情報収集すると、罠情報を取得します。
- 罠情報はスパイ側では本物と同じ色で表示されます。

## 現在の注意点

- `OFFLINE`では1台のPCで片方のロールを操作します。オーガナイザー選択時だけ対戦相手のテスト用スパイAIが動きます。
- `Esc` は端末メニューのキャンセルではなく、マウスカーソルのロック切り替えです。端末メニューは `E` でキャンセルします。
- 現在のシーンは添付間取りを基に複雑化した3階施設です。全階のNPC巡回と偽造命令への反応まで実装済みで、オンライン対戦はまだ実装されていません。
