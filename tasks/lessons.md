# Lessons

- CharacterControllerの開始地点は足元を床面と完全一致させず、少し上へ置く。施設再生成処理、シーン保存値、AI開始値を同じ座標へ統一し、プレイ領域外への落下復帰も用意する。

- Unityで実行時生成されるオブジェクトを案内するときは、Play前のHierarchyには存在しないこと、Play開始後に生成されること、確認するタブまで明記する。
- 操作ガイドでは「何を押すか」だけでなく、操作可能になる前提状態（Playモード、Gameビュー、カーソルフォーカス）も手順に含める。
- 実行時生成したUnity UIのクリックをInput Moduleだけに依存させず、主要な選択にはキーボード入力または直接ポインター判定のフォールバックを用意する。
- 正方形のRenderTextureは画面全体へストレッチせず、AspectRatioFitterなどで縦横比を維持する。
- 完成済みロゴの形状・文字品質が重要な場合は、再構成や生成を提案する前に「原画をそのまま使うか」を確認する。厳密な保持指定では透過Spriteを唯一の表示ソースにし、動きは別レイヤーへ限定する。
- UI用原画の文字品質を保つ場合、Texture ImporterのMax Sizeは原画の最大辺以上にし、意図しないインポート時縮小を避ける。
- 監視ゲームで情報制限を作るときは、カメラを真上から配置せず、壁高を天井まで確保した透視の室内・通路視点で死角を残す。
- カメラ一覧の「選択」は枠の強調だけで終わらせず、必ず詳細表示の映像ソースまで同じ選択状態へ連動させる。クリック対象もセル全体に用意する。
- 管理役のゲーム開始を即座に全画面端末UIへ固定せず、まず操作可能な空間へ置き、端末への接近・注視・操作を経て情報画面へ入れる。端末を離れたら表示も切る。
- 複雑な複数階施設は同じ四分割テンプレートを積層せず、階ごとの目的、中央動線、認証区画、監視死角を先に定義してから固有の間取りとして組み立てる。
- 1階だけを俯瞰表示する場合はカメラ高度だけに頼らず、専用レイヤーとCulling Maskで上階による遮蔽と情報漏洩を同時に防ぐ。
- 潜入ステージでは操作端末だけを置くと目的物が即座に判別できるため、同じシルエットのブラフPCを執務空間へ十分な密度で混ぜ、操作プロンプトでのみ識別させる。
- 大型化では床と外壁だけを拡張せず、部屋座標、ドア、端末、エレベーター、監視カメラ、検知半径、俯瞰カメラ範囲を同じ平面倍率へ追従させる。
- Screen Space Overlay UIだけを表示し、全Display向けカメラを無効にするとGameビューへ`No cameras rendering`が出る。RenderTexture専用カメラでは代替できないため、Culling Mask 0の最背面カメラをDisplayへ残す。
- NavMesh巡回はスポーン地点がNavMesh上にあるだけでは不十分で、ルートの全区間を`NavMesh.CalculatePath`し、`PathComplete`まで自動検証する。
- `NavMeshSurface.BuildNavMesh()`で生成したデータをシーンへ直接埋めるとバイナリ化する場合がある。NavMeshDataを独立した`.asset`へ保存し、シーンはForce Textで維持する。
- 複数階のNavMeshは階ごとに独立したアセットへ分離し、NPCと命令対象にも階番号を持たせる。同一XZ座標に別階が重なるため、発行端末と同じ階だけを反応対象にする。
- 偽造命令によるNPC移動は命令ログのイベントを起点にし、時間切れと無効化の両方で通常巡回へ戻せる一時状態として実装する。
- 遅延映像からゲーム判定を行う場合、表示時点の現在座標を使ってはいけない。映像リングバッファと同じ書き込み位置へ撮影時点の判定情報も保存し、読み出しインデックスを共有する。
- 一人称キャラクターの外見を監視カメラへ映す場合、Rendererを削除してはいけない。自己表示専用レイヤーを一人称カメラだけから除外し、外部カメラには同じ外見を描画する。
- ブラフ端末は単なる操作不能オブジェクトより、通常業務という安全な操作を持たせた方が、本物の端末を隠しながらプレイヤーへ擬態行動の選択肢を与えられる。
- Unityでシーンに保存する`MonoBehaviour`は、publicクラス名とファイル名を一致させて独立ファイルに置く。別クラスのファイルに同居させると、追加直後は動いてもシーン再読み込み後に復元できない。
- 施錠室のようにNavMeshが分離する場所は、経路計算だけでは横断できない。認証・ハッキングなどの明示的な状態遷移と、移動先NavMeshの個別検証を用意する。
- URPのRenderTextureを使うPlay Mode自動検証は、`-nographics`ではレンダリングエラーによりゲーム進行も止まることがある。GPU有効のバッチ検証で実際の進行を確認する。
- リングバッファを再生成するときは配列だけでなく、書き込み位置、有効件数、経過タイマーも同時にリセットする。サイズ変更後に旧カーソルが残ると配列外参照になる。
- オンライン監視ゲームではRenderTexture自体を配信せず、座標や状態を同期して受信側のカメラで再描画する。セッションメタデータはNetworkObjectを必要としないCustom Messageで先に縦切りできる。
