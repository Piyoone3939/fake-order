# Lessons

- Unityで実行時生成されるオブジェクトを案内するときは、Play前のHierarchyには存在しないこと、Play開始後に生成されること、確認するタブまで明記する。
- 操作ガイドでは「何を押すか」だけでなく、操作可能になる前提状態（Playモード、Gameビュー、カーソルフォーカス）も手順に含める。
- 実行時生成したUnity UIのクリックをInput Moduleだけに依存させず、主要な選択にはキーボード入力または直接ポインター判定のフォールバックを用意する。
- 正方形のRenderTextureは画面全体へストレッチせず、AspectRatioFitterなどで縦横比を維持する。
- 完成済みロゴの形状・文字品質が重要な場合は、再構成や生成を提案する前に「原画をそのまま使うか」を確認する。厳密な保持指定では透過Spriteを唯一の表示ソースにし、動きは別レイヤーへ限定する。
- UI用原画の文字品質を保つ場合、Texture ImporterのMax Sizeは原画の最大辺以上にし、意図しないインポート時縮小を避ける。
